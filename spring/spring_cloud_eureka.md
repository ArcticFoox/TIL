# spring cloud eureka
서비스 레지스트리와 서비스 디스커버리를 지원하기 위한 라이브러리

## 주요 개념
- 서비스 레지스트리: 서비스 인스턴스의 위치(호스트, 포트 등)를 저장하는 중앙 저장소
- 서비스 디스커버리: 클라이언트가 서비스 인스턴스를 동적으로 찾고 연결하는 메커니즘 
### Eureka Server
- REST API 기반으로 다양한 언어에서 사용 가능
- 레지스트리의 모든 정보는 모든 Eureka Client에서 복제되어 있으며 가용 중인 모든 서비스 목록 확인 가능 및 30초 마다 갱신
- Client들에게 자신이 가지고 있는 Client 목록 정보 공유(Gateway, Ribbon, Zipkin 등에서 사용)
- 일정 간격으로 Client로 부터 핑을 전달 받아 가용 상태를 확인 특정 횟수 이상 핑이 안되면 Client 목록에서 제외
- 가용 상태의 서비스 목록 확인 시 서비스의 이름을 기준으로 탐색, 로드 밸런싱을 위해 내부적으로 Ribbon 사용

### Eureka Client
- 서비스 시작 시 Server에 자신을 등록
- 주기적으로(30초가 기본) Server에 자신이 살아 있음을 알림(Heartbeat)
- 레지스트리로부터 다른 Client 서비스 정보를 확인할 수 있음
