# JIB build
##빠른 애플리케이션 빌드 속도
### 재생 가능한 빌드 방식
동일한 컨텐츠로 컨테이너 이미지를 다시 빌드하면 항상 동일한 이미지 생성
불필요한 업데이트가 다시 실행되지 않으며, 변경된 내용이 없다면 이미지 빌드 과정이 수행되지 않음
### Docker Daemonless
도커 데몬으로부터 자유로워 CLI 종속성을 줄임
Dockerfile을 작성하고 build/push 호출 안해도 됨
간단히 Gradle, Maven 내에서 Docker 이미지, 레지스 트리, 로그인 정보를 작성하여 처리
  
<br>

# JIB vs Dockerfile
JIB 애플리케이션의 구조를 인식해서 불필요한 빌드를 최소화
Dockerfile은 모든 파일을 통째로 COPY 하거나 RUN 하는 경우, 레이어별 캐싱이 어려움


| 항목     | JIB                         | Dockerfile            |
|--------|-----------------------------|-----------------------|
| 빌드 방식  | java 레벨에서 직접 레이어 생성         | Docker 데몬을 통해 이미지 조립  |
| 의존성 관리 | 클래스/리소스/의존성 레이어 자동 분리       | 수동으로 COPY 해야 최적화됨     |
| 캐시     | Gradle/Maven 캐시 + JIB 자체 캐시 | Docker 레이어 캐시         |
| 설정     | Gradle/Maven 플러그인으로 설정 가능      | 보통 JAR 빌드 후 COPY 해야 함 |
