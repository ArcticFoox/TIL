# JPA 연관관계

연관관계 매핑의 핵심은 방향성, 다중성, 주인 설정

방향성은 단방향과 양방향으로 나뉨

다중성은 1:1, 1:N, N:1, N;N 관계를 포함

연관관계 주인은 데이터베이스의 외래 키를 관리하는 객체를 의미

주인을 올바르게 설정하지 않으면 데이터에비스에 불필요한 조인 테이블이 생성되거나, 예상치 못한 쿼리 실행

### 주의사항

양방향 관계에서는 연관관계 주일을 명확히 설정해야 함

주인을 잘못 설정하면 불필요한 조인 테이블이 생성되거나, 데이터 무결성 문제가 발생할 수 있음

Cascade 옵션과 Orhan Removal 옵션을 사용할 때 신중해야 함

Cascade는 부모 엔티티의 상태 변화가 자식 엔티티에 전파되도록 설정

Orphan Removal은 부모와 자식을 자동으로 삭제

Fetch 전략을 설정

Lazy Loading은 실제로 데이터를 사용할 때 쿼리 실행

Eager Loading은 연관된 데이터를 한 번에 가져옴
