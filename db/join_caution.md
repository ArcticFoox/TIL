# JOIN 주의점
## 사고흐름
"누가 조회하느냐"와 "무엇이 조회 대상이냐"를 혼동하면 안됨  
요구사항을 SQL Join 순서로 바로 번역하는 실수

## 우선 목표
**무엇이 조회 대상인가?**  
-> SELECT 대상이 무엇인가?  
JOIN의 기준 축(anchor)은 "조회 결과의 주체"

### 체크리스트
- [ ] SELECT 컬럼 중 가장 중요한 테이블이 무엇인가?
- [ ] FK가 직접 연결되어 있는가?
- [ ] 이 JOIN을 WHERE / EXISTS로 바꿀 수 있는가?
- [ ] 쿼리를 한글로 번역했을 때 자연스러운가?
- [ ] DISTINCT가 필요한가? 필요하다면 의심하자
- [ ] 다대다 테이블을 메인 JOIN 축으로 삼지 않았나?
- [ ] JOIN 경로가 FK를 우회하지 않았나?

## 요약
JOIN은 데이터를 가져오는 도구고,
권한/소속은 필터
필터를 기준 축으로 삼지 말자
