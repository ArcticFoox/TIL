# Kubernetes Helm
Helm은 Kubernetes 애플리케이션의 패키지 관리자 역할을 하는 도구로, 복잡한 Kubernetes 리소스 구성을 간소화하고 재사용 가능한 패키지 형태로 배포할 수 있도록 지원  
Helm을 사용하면 애플리케이션의 설치, 업그레이드, 삭제를 쉽게 관리할 수 있으며, Helm 차트(Chart)를 통해 애플리케이션의 모든 구성 요소를 정의할 수 있음  

## Helm의 주요 개념
- 차트(Chart): Kubernetes 리소스의 템플릿과 설정 파일을 포함하는 패키지. 애플리케이션의 구성 요소를 정의하고 버전 관리를 지원  
- 릴리스(Release): 특정 차트를 기반으로 생성된 Kubernetes 리소스스의 인스턴스. 동일한 차트를 여러 번 설치하여 여러 릴리스를 생성할 수 있음  
- 저장소(Repository): 차트를 호스팅하는 장소. 공용 또는 사설 저장소를 통해 차트를 공유하고 배포할 수 있음
- 값 파일(Values File): 차트의 템플릿에 전달되는 사용자 정의 설정을 포함하는 파일. 기본값을 재정의하여 다양한 환경에 맞게 배포 가능
- 템플릿(Template): 차트 내에서 Kubernetes 리소스 정의를 동적으로 생성하기 위한 Go 템플릿 언어를 사용한 파일

## Helm의 주요 기능
- 간편한 설치 및 업그레이드: `helm install` 및 `helm upgrade` 명령어를 통해 애플리케이션을 쉽게 설치하고 업그레이드할 수 있음  
- 버전 관리: 차트와 릴리스의 버전을 관리하여 특정 시점으로 롤백하거나 특정 버전을 배포할 수 있음  
- 의존성 관리: 차트 간의 의존성을 정의하고 자동으로 설치 및 업데이트할 수 있음

### 이해한 내용
차트를 구성하는 방법 중 Artifact Hub에서 공개된 차트를 활용하는 방법과 직접 차트를 만들어 사용하는 방법이 있음  
공개된 차트를 활용하는 방법은 이미 잘 만들어진 차트를 사용하여 빠르게 애플리케이션을 배포할 수 있는 장점이 있음
직접 차트를 만드는 방법은 애플리케이션의 특성에 맞게 세밀하게 조정할 수 있는 장점이 있음

직접 차트를 만드는 경우 차트 디렉토리 구조를 이해하는 것이 중요함
- Chart.yaml: 차트의 메타데이터를 정의하는 파일
- values.yaml: 차트의 기본 설정 값을 정의하는 파일
- templates/: Kubernetes 리소스 정의를 포함하는 템플릿 파일들이 위치하는 디렉토리
- charts/: 차트의 의존성을 포함하는 디렉토리

`chart create` 명령어로 기본 차트 구조를 생성한 후, `values.yaml` 파일을 수정하여 애플리케이션의 설정 값을 정의할 수 있음
